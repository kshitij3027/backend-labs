# Backend Labs

A collection of independent mini-projects built during a backend engineering learning journey.

## Projects

| Project | Description | Tech |
|---|---|---|
| [configurable-log-generator](./configurable-log-generator/) | Continuously generates realistic, timestamped application logs at a configurable rate with burst mode and state machine patterns | Python 3.12, Docker |
| [log-capture-service](./log-capture-service/) | Real-time log file watcher that parses text/JSON logs, applies regex filtering and tagging, and writes structured JSON batches | Python 3.12, watchdog, Docker |
| [log-parsing-service](./log-parsing-service/) | Transforms raw log lines (Apache, Nginx, JSON, Syslog) into normalized structured JSON with auto-format detection and aggregate statistics | Python 3.12, watchdog, Docker |
| [log-storage-service](./log-storage-service/) | File-based log storage engine with automatic rotation, compression, and purging based on configurable policies, plus a CLI log inspector | Python 3.12, Docker Compose |
| [log-query-cli](./log-query-cli/) | CLI tool that parses, filters, and searches log files using a memory-efficient generator pipeline with text/JSON/color output and statistics | Python 3.12, Docker |
| [log-processing-pipeline](./log-processing-pipeline/) | End-to-end pipeline that generates, collects, parses, stores, and queries log data using five Docker containers with shared volumes | Python 3.12, Docker Compose |
| [tcp-log-collection-server](./tcp-log-collection-server/) | Multi-threaded TCP server that receives NDJSON log messages, filters by level, persists to disk, and rate-limits per client IP | Python 3.12, Docker Compose |
| [log-shipping-client](./log-shipping-client/) | TCP log shipping client that reads log files, formats as NDJSON, and ships over TCP with compression, batching, reconnect, health monitoring, and metrics | Python 3.12, Docker Compose |
| [udp-log-shipping-system](./udp-log-shipping-system/) | UDP-based log forwarder — client ships JSON logs over UDP, server buffers and persists to disk with priority handling, ACKs, Flask dashboard, and load testing | Python 3.12, UDP sockets, Flask, Docker |
| [batch-log-shipper](./batch-log-shipper/) | UDP-based batch log shipper — client collects logs into configurable batches with compression, splitting, and retry before shipping to a UDP server | Python 3.12, Docker Compose |
| [log-compression-system](./log-compression-system/) | Client-server log shipping system with batched gzip/zlib compression over TCP, adaptive compression levels, wire protocol framing, and stats dashboard | Python 3.12, TCP sockets, gzip/zlib, psutil, Docker Compose |
| [tls-encrypted-log-transmission](./tls-encrypted-log-transmission/) | TLS 1.3 encrypted log transmission with gzip compression, self-signed certs, healthcare simulation with HIPAA anonymization, log rotation, and metrics dashboard | Python 3.12, TLS/SSL, Docker Compose |
| [universal-log-translator](./universal-log-translator/) | Plugin-based log normalizer that auto-detects and translates JSON, plain text/syslog, Protobuf, and Avro into a standardized LogEntry with adaptive performance tracking | Python 3.12, Click, protobuf, fastavro, Docker |
| [schema-registry-service](./schema-registry-service/) | Centralized REST API that stores, versions, and validates log message schemas with SHA-256 deduplication, compiled validator caching, and backward compatibility checking | Python 3.12, Flask, jsonschema, fastavro, Docker |
| [log-format-compatibility-layer](./log-format-compatibility-layer/) | Auto-detect, parse & translate logs from syslog/journald/JSON into a unified schema | Python, Flask, Click, Docker |
| [adaptive-batching-engine-optimizer](./adaptive-batching-engine-optimizer/) | Self-tuning batch-size optimizer: gradient-ascent control loop (FastAPI + WebSocket) with a live Chart.js dashboard | Python, FastAPI, NumPy, psutil, WebSocket, Docker |
| [multi-tier-caching-layer](./multi-tier-caching-layer/) | Read-through multi-tier query cache (L1 in-memory → L2 Redis → L3 Postgres materialized → slow backend) with heuristic pattern-learning, proactive warming, zstd compression, and a live Chart.js/WebSocket dashboard | Python 3.12, FastAPI, Redis, PostgreSQL, asyncpg, Docker Compose |
| [storage-format-optimizer](./storage-format-optimizer/) | Adaptive log storage engine that learns query access patterns and auto-migrates partitions between row, columnar (Parquet), and hybrid formats to optimize latency + size; copy-on-write migration with atomic manifest swaps and a live WebSocket dashboard | Python 3.12, FastAPI, PyArrow, Docker |
| [bloom-filter-log-membership](./bloom-filter-log-membership/) | Per-log-type + sessions Bloom membership filters answering "have we seen this log entry before?" with sub-millisecond lookups, zero false negatives, and tunable false-positive rates; scalable adaptive sizing, daily rotation generations, crash-safe CRC-validated snapshots, a two-tier SQLite lookup pipeline with FP-threshold fallback, and a separate real-time WebSocket dashboard process | Python 3.12, FastAPI, mmh3, bitarray, SQLite, WebSocket, Docker |
| [delta-encoding-log-engine](./delta-encoding-log-engine/) | Delta-encoding log storage engine (keyframe + field-level deltas with typed int/string encoders, ~70% lossless reduction beating gzip structurally and composing with it) plus a live FastAPI WebSocket dashboard on :8080 | Python 3.12, FastAPI, Chart.js, Docker Compose |
| [adaptive-resource-allocation-log](./adaptive-resource-allocation-log/) | Self-optimizing log-processing autoscaler: psutil metrics + Holt load forecasting drive reactive+predictive worker scaling (hysteresis, cooldowns, anomaly detection, time-of-day patterns, cost reporting) with a live SocketIO/Chart.js dashboard on :8080 | Python 3.11, Flask, Flask-SocketIO, eventlet, psutil, NumPy, Docker Compose |
| [ml-log-classifier](./ml-log-classifier/) | Streaming ML pipeline that classifies raw logs by severity + category in real time via a soft-voting ensemble (MultinomialNB+RandomForest+GradientBoosting over TF-IDF+temporal+metadata features); FastAPI REST/stream/WebSocket API with on-demand training + zero-downtime hot-swap, hierarchical multi-service classification + anomaly scoring, adaptive drift retraining, A/B serving with graceful fallback, prediction caching, and a live React+Chart.js dashboard on :8080 | Python 3.11, FastAPI, scikit-learn, NLTK, React+Vite+Chart.js, nginx, Docker Compose |
